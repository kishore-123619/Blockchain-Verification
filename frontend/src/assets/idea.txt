To connect blockchain in your MERN project, you mainly connect **Frontend ↔ Wallet ↔ Smart Contract ↔ Blockchain**.

Since you are using:

* React (Frontend)
* Node.js + Express (Backend)
* MongoDB
* Ethereum / Sepolia
* Solidity
* MetaMask
* Ethers.js

this is the standard architecture:

## Overall Flow

```text
React Frontend
      ↓
MetaMask Wallet
      ↓
Ethers.js
      ↓
Smart Contract (Solidity)
      ↓
Ethereum / Sepolia Blockchain

Backend (Node.js + MongoDB)
      ↓
Stores user data, logs, certificates, etc.
```

# Step-by-Step Blockchain Connection

# 1. Install Packages

Inside frontend:

```bash
npm install ethers
```

Inside backend/project root:

```bash
npm install hardhat dotenv
```

---

# 2. Create Smart Contract

Example Solidity contract:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract Certificate {
    string public message;

    function setMessage(string memory _msg) public {
        message = _msg;
    }

    function getMessage() public view returns(string memory){
        return message;
    }
}
```

Save inside:

```text
contracts/Certificate.sol
```

---

# 3. Compile & Deploy Contract

Use [Hardhat](https://hardhat.org?utm_source=chatgpt.com)

Install:

```bash
npm install --save-dev hardhat
```

Initialize:

```bash
npx hardhat
```

Deploy to Sepolia using:

* MetaMask private key
* RPC URL from:

  * [Alchemy](https://www.alchemy.com?utm_source=chatgpt.com)
  * [Infura](https://www.infura.io?utm_source=chatgpt.com)

Example `.env`:

```env
SEPOLIA_RPC_URL=your_rpc_url
PRIVATE_KEY=your_wallet_private_key
```

Deploy command:

```bash
npx hardhat run scripts/deploy.js --network sepolia
```

After deployment you get:

```text
Contract Address
```

Example:

```text
0x123abc....
```

# 4. Connect MetaMask in React

## Frontend Wallet Connection

```javascript
import { ethers } from "ethers";

async function connectWallet() {
  if (window.ethereum) {

    await window.ethereum.request({
      method: "eth_requestAccounts",
    });

    const provider = new ethers.BrowserProvider(window.ethereum);

    const signer = await provider.getSigner();

    const address = await signer.getAddress();

    console.log(address);
  } else {
    alert("Install MetaMask");
  }
}
```

This connects:

```text
React → MetaMask
```

---

# 5. Connect Smart Contract

## Add ABI + Contract Address

After deployment Hardhat gives ABI.

Create:

```text
src/contract.js
```

```javascript
import { ethers } from "ethers";

const contractAddress = "YOUR_CONTRACT_ADDRESS";

const abi = [
  "function setMessage(string memory _msg) public",
  "function getMessage() public view returns(string memory)"
];

export async function getContract() {

  const provider = new ethers.BrowserProvider(window.ethereum);

  const signer = await provider.getSigner();

  const contract = new ethers.Contract(
    contractAddress,
    abi,
    signer
  );

  return contract;
}
```

---

# 6. Write Data to Blockchain

```javascript
const contract = await getContract();

const tx = await contract.setMessage("Hello Blockchain");

await tx.wait();

console.log("Stored");
```

Flow:

```text
React → MetaMask → Smart Contract → Blockchain
```

# 7. Read Data from Blockchain

```javascript
const contract = await getContract();

const message = await contract.getMessage();

console.log(message);
```

# 9. Real Certificate Verification Flow

## Admin Uploads Certificate

### Flow

```text
1. Admin uploads certificate
2. Backend generates SHA256 hash
3. Hash stored on blockchain
4. Original file stored in MongoDB/cloud
5. Verification checks hash match
```

---

# 10. What Actually Goes to Blockchain?

DO NOT store full PDFs on blockchain.

Store only:

```text
Certificate Hash
Student ID
Timestamp
Transaction ID
```

Because blockchain storage is expensive.

# Best Practice

Use blockchain only for:

* Verification
* Ownership
* Immutable records
* Hash storage

Use MongoDB for:

* User data
* Large files
* Fast queries
* Authentication
